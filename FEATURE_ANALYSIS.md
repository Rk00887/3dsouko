### 実装アプローチ

**プロジェクト管理（F-02〜F-09）**

```
現行: LayoutManager (localStorage)
↓
拡張: ProjectManager
  - project: { id, name, description, createdAt, updatedAt, layoutData }
  - IndexedDB または バックエンドDB（NestJS）に保存
  - エクスポート: JSON.stringify → Blob → ダウンロード
  - インポート: FileReader → JSON.parse → LayoutManager.load()
```

**図面ダウンロード（F-14〜F-16）**

```
Three.js renderer.domElement.toDataURL('image/png')
→ PNG/JPEG はそのまま出力可能
→ PDF は jsPDF ライブラリ使用 (html2canvas + jsPDF)
```

**キーボードショートカット拡張（F-12）**

| ショートカット | 動作 |
|---|---|
| Ctrl+S | 保存 |
| Ctrl+N | 新規プロジェクト |
| Ctrl+D | 複製（現在未実装） |
| ← ↑ → ↓ | オブジェクト移動 |
| R | 回転モード（実装済み） |
| Delete / Backspace | 削除（実装済み） |
| Ctrl+Z / Ctrl+Y | Undo/Redo（実装済み） |

## 機能分析：商品マスタ

商品マスタとは**SKU（商品単位）の管理・検索・WMS連携**機能カテゴリ。

### 機能一覧と分析

| # | 機能名 | 詳細 | 現行 | 実装難易度 | 優先度 |
|---|---|---|---|---|---|
| M-01 | 商品登録 | SKU名・寸法・重量・箱数登録 | ❌ 未実装 | ★★☆ | 高 |
| M-02 | 商品編集 | 既存SKU情報の更新 | ❌ 未実装 | ★★☆ | 高 |
| M-03 | 商品検索 | SKUコード・名前での検索 | ❌ 未実装 | ★★☆ | 高 |
| M-04 | 商品一覧 | 登録済みSKU一覧表示 | ❌ 未実装 | ★★☆ | 高 |
| M-05 | WMS連携インポート | WMSから商品マスタ一括取込 | ❌ 未実装 | ★★★★ | 低（後回し） |

### データモデル

```
sku: {
  id, skuCode, name,
  width, depth, height,  // 個品寸法 (m)
  weight,                // 重量 (kg)
  boxQty,                // 1箱あたり個数
  boxWidth, boxDepth, boxHeight, boxWeight  // 箱寸法
}
```

### 実装アプローチ

現行は商品マスタが存在せず、箱オブジェクトは直接寸法入力のみ。

```
SKUDialog（モーダル）
  ├── SKU一覧表示
  ├── SKU登録・編集フォーム
  └── SKU検索バー

箱オブジェクト追加時:
  Option A: SKUを選択して箱を生成（推奨）
  Option B: 直接入力（既存の動作を維持）
```

DESIGN.md のSKUドメイン設計（`sku`, `carton` テーブル）が既に存在しており、実装ガイドとして活用可能。

### 詳細分析

**コピー（OP-01）**

```
実装方針:
  Ctrl+D または右クリックメニュー → 選択オブジェクトをClone
  userData をディープコピー → buildXxx() で新規オブジェクト生成
  スポーン位置: 元オブジェクトから +0.5m オフセット（衝突解決付き）
```

**矢印キー移動（OP-08）**

```
現行の _onKeyDown で R/Del のみ対応
↓
ArrowUp / ArrowDown / ArrowLeft / ArrowRight を追加
移動量: 0.5m（グリッドサイズ）or Shift押下で 0.1m
衝突検出を通じて moveToXZ() 経由で移動
```

**新規オブジェクト種別の実装方針**

| 種別 | 実装方法 |
|---|---|
| 中量ラック (O-02) | buildRack() に `variant: 'medium'` パラメータ追加（支柱細め・棚板薄め） |
| 柱 (O-10) | buildPillar() 新規追加（CylinderGeometry or BoxGeometry） |
| ドア (O-11) | buildDoor() 新規追加（壁面埋め込み型・独立型） |
| 通路 (O-12) | buildAisle() 新規追加（PlaneGeometry + 半透明マテリアル） |
| 作業台 (O-08) | buildWorkbench() 新規追加 |
| ベルトコンベア (O-06) | buildConveyor() 新規追加（帯 + ローラー繰り返し） |
| フォークリフト (O-09) | GLTF モデル or buildForklift() 簡易プロシージャル |

**グループ機能（OP-10）**

```
実装難度: 高（Three.js Group + userData.type='group' + 全子要素一括操作）
アプローチ:
  - Shift+クリック で複数選択
  - Ctrl+G でグループ化 → THREE.Group に追加
  - グループ選択時、グループ全体にTransformControls適用
  - グループ解除: Ctrl+Shift+G
```

## 機能分析：3D化

3D化とは**2D図面・データを3D空間に変換・可視化する**機能カテゴリ。

### 機能一覧と分析

| # | 機能名 | 詳細 | 現行 | 実装難易度 | 優先度 |
|---|---|---|---|---|---|
| T-01 | フレーム3D化 | 倉庫建物（壁・天井・床）の3D表示 | △ 床・グリッドのみ | ★★★ | 高 |
| T-02 | オブジェクト3D化 | 全オブジェクト種別の3Dモデル化 | △ 一部のみ | ★★★ | 高 |
| T-03 | コンテナ計算3D化 | 3D装箱アルゴリズムの3D可視化 | ❌ 未実装 | ★★★★ | 中 |

### 詳細分析

**フレーム3D化（T-01）**

```
現行: 床面（PlaneGeometry）+ GridHelper のみ

追加が必要:
  - 倉庫壁面（4面 + 開口部）
  - 天井（オプション表示/非表示）
  - 倉庫寸法設定UI（幅・奥行・高さ・天井高）
  - 建物輪郭のシルエット強調

実装方針:
  buildWarehouseFrame({ width, depth, height, wallThickness })
  → 4壁 + 床 + 天井オプション
  → 倉庫新規作成時にダイアログで寸法入力
```

**コンテナ計算3D化（T-03）**

DESIGN.md の「3D装箱アルゴリズム」と直結する機能。

```
入力: 棚/パレットを選択 + SKUを指定
↓
3D Bin Packing 計算（floor division + 回転最適化）
↓
結果を3D空間に可視化:
  - パレット/棚の内部に箱を積み上げる形で表示
  - 充填率・箱数・重量をオーバーレイ表示
```
